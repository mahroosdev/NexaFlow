# NexaFlow v1.0 Publication Checklist

## Release Scope

- Windows Desktop v1.0.0
- Android Companion v1.0.0 (`com.nexaflow.mobile`, version code `101`)
- Chrome Companion v1.0.0
- Static website, privacy policy, support page, and checksums

## Website Files

Netlify publishes `website/` directly, or the root-level `website.zip` generated by `tools/prepare-public-release.ps1`:

- `website/index.html`
- `website/downloads.html`
- `website/support.html`
- `website/privacy.html`
- `website/release-checksums-sha256.txt`
- `website/site-data/releases.js`
- `website/assets/`, `website/favicon.ico`, and `website/robots.txt`
- `website/_redirects` and `website/_headers`

## Release Assets

- Public Windows package: `App Versions/App Version 1.0.0/NexaFlow_Windows_v1.0.0.zip` (setup EXE and PDF guide)
- Public Android APK: `App Versions/App Version 1.0.0/NexaFlow_Android_v1.0.0.apk`
- Public Chrome ZIP: `App Versions/App Version 1.0.0/NexaFlow_Chrome_Extension_v1.0.0.zip`
- Play Console AAB: `mobile-app/Android Installer/NexaFlow_Android_Store.aab`
- Secondary Windows setup ZIP: `windows-setup/Installers/NexaFlow_v1.0_Setup.zip`

The AAB is for Google Play Console only. The APK is the direct-install Android file. Chrome uses a manual extension ZIP and does not have an installer.

## Automated Gate

- Run `scripts/validate-public-repo.ps1`.
- Run Python compilation and all tests in `tests/`.
- Run both mobile Node test files and the Chrome recording-safety test.
- Confirm all five SHA-256 values match the release files.
- Confirm the root and website checksum files are identical.
- Confirm the Android package, version, target SDK, cleartext flag, and signing certificate.

## Manual Product Gate

- Run the Windows app from source and from the packaged standalone executable.
- Confirm the NexaFlow lightning icon in the taskbar, main window, Focus window, and pairing dialog.
- Start Remote Access repeatedly and confirm a valid firewall rule is reused without another prompt.
- Install the APK on a real phone and approve pairing over the same Wi-Fi or hotspot.
- Confirm automatic trusted reconnect and Clear Pairing behavior.
- Choose a workflow and test countdown, Start, Pause, Resume, Stop, repeat, and loop modes.
- Confirm the desktop viewer retains its previous frame during slow refreshes.
- Test Sharp, Balanced, and Battery quality presets.
- Test full-screen landscape viewing, pinch zoom, pan, double-tap reset, and close.
- Load the canonical `chrome-extension/` folder in Chrome developer mode and test pairing, recording, export/import, and playback.

## Publication Gate

- Upload the three public packages in `App Versions/App Version 1.0.0/` to a release-file host.
- Set `releaseBase` in `website/site-data/releases.js` to that host's direct-download base URL, then set `published` to `true`.
- Deploy `website.zip` to Netlify and verify `/downloads`, `/support`, and `/privacy`.
- Submit the AAB separately through Google Play Console when the store listing is ready.
- Add Google Play and Chrome Web Store URLs only after the listings are approved.
- Keep the Android release keystore and credentials backed up outside the repository.
- Expect Windows SmartScreen warnings until the installer is code-signed.

## Known Follow-Up

- Expo SDK modernization is separate from v1 because it requires a major dependency migration and another full Android regression pass.
- The final physical-phone UI check remains mandatory before calling the Android release hardware-verified.

_Scope: v1.0 targets Windows Desktop and Android, with the Chrome companion._
