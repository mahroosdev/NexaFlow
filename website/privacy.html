Privacy Policy

NexaFlow is designed around local workflows, visible pairing, and user-controlled support submissions.

Scope

This policy covers NexaFlow Desktop for Windows, the NexaFlow Android companion, the NexaFlow Chrome extension, this website, and the support process. NexaFlow is developed and published by Mahroos Mahthie.

Data stored locally

NexaFlow can store workflows, schedules, settings, recent-file references, screenshots, trusted-device records, and browser workflow data on the user's devices. This information is not automatically uploaded to the developer.

The Chrome extension excludes password-field values from recorded click labels and workflow data. Users should still review exported workflows before sharing them.

Android pairing and Remote Access

Remote Access is off by default. The desktop user starts it manually or enables Start Remote when NexaFlow opens. A new Android device sends a named pairing request, and the desktop user approves or denies it. Trusted devices can reconnect until trust is revoked.

The Android companion and Windows desktop communicate directly over the local network using unencrypted HTTP. This may include playback status, desktop screen images, workflow selections, and approved commands. Use NexaFlow only on a trusted private Wi-Fi network or a phone hotspot. Do not expose port 8765 through router port forwarding or the public internet.

The Chrome extension connects to NexaFlow Desktop through unencrypted loopback HTTP on the same Windows PC. It does not require LAN access. Chrome pairing and commands still require desktop approval.

Retention and deletion

Local data remains until the user deletes files, clears recent data, clears pairing, revokes a trusted device, removes the extension, clears application storage, or uninstalls the application. Exported files and screenshots remain wherever the user saved them until manually deleted.

Website and downloads

The static website does not include advertising, account registration, behavioral analytics, or an email collection service. The hosting provider may process standard request information such as IP address, browser details, requested files, timestamps, and security logs under its own terms.

Large application packages may be served by an external release host. That provider may process download request information under its own privacy policy.

Support submissions

The support form prepares a public support ticket. Nothing is submitted until the user continues to the issue tracker and confirms it. The issue tracker's hosting provider processes the account and ticket information under its own terms. Users must remove passwords, pairing tokens, private IP addresses, screenshots, and workflow data before submitting.

Security choices

  • Keep Remote Access off when it is not needed.
  • Approve only devices you recognize.
  • Use a trusted private network or personal hotspot.
  • Revoke pairing if a phone is lost, sold, or shared.
  • Verify release checksums before direct installation.

Policy changes

Material policy changes will be published on this page with a revised effective date. The current policy applies to NexaFlow v1.0.0.

Privacy questions

Use the NexaFlow Support page to prepare a support ticket. Do not include confidential information because submitted tickets are normally public.
