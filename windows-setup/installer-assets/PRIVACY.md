# NexaFlow Privacy

NexaFlow is designed as a local-first desktop automation application with optional user-approved companion access.

## Local Data

NexaFlow may store workflows, browser workflows, preferences, pairing settings, and recent-file settings on the user's local device. These files are used to run and manage automation workflows.

## Network Use

The application package does not require a cloud account for normal local operation.

Remote Access is off by default. The desktop user can start it manually or enable **Start Remote when NexaFlow opens** to start it automatically on future launches. Approved Android companions connect over a trusted local Wi-Fi network or phone hotspot. The Chrome extension connects through loopback HTTP on the same Windows PC. NexaFlow does not include telemetry, analytics, or cloud account login in this v1.0 package.

NexaFlow does not provide silent remote access. Pairing requests expire, new devices require desktop approval, and the desktop user can stop Remote Access at any time.

## Support Data

The website support page prepares a public GitHub issue. If you continue to GitHub and submit it, the issue text and associated GitHub account information are processed by GitHub and are normally public. Review the issue before submission and do not include passwords, pairing tokens, screenshots, or sensitive workflow data.

## User Responsibility

Automation workflows can contain actions, text input, clipboard use, or paths created by the user. Review workflows before sharing them.
